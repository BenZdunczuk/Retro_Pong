◆ accel_calibrate()

void accel_calibrate ( int16_t offset_cb[3],
uint16_t samples )

Funkcja kalibrująca działanie akcelerometru.

Oblicza średnie przesunięcie (offset) dla każdej osi akcelerometru:

  • pobiera samples kolejnych odczytów,
  • sumuje je i dzieli przez liczbę próbek,
  • zapisuje wynik do bufora offset_cb[] i do wewnętrznego acc_offset[].
Parameters
[in]offset_cb[3]tablica, do której zwracane są zmierzone offsety,
[in]samplesliczba próbek użytych do obliczenia średniej.

◆ accel_lpf_int()

int16_t accel_lpf_int ( int16_t in,
int16_t * prev,
uint8_t alpha )

Funkcja filtrująca dane z akcelerometru.

Integer-owy filtr dolnoprzepustowy IIR: y[n] = y[n-1] + α·(x[n] − y[n-1]), gdzie α = alpha/256. Używa wskaźnika prev, by zaktualizować i zachować stan filtra pomiędzy wywołaniami.

Parameters
[in]inbieżąca surowa, już wycentrowana wartość,
[in]prevwskaźnik na poprzednią wartość filtra (i miejsce przechowywania wyniku),
[in]alphawspółczynnik filtra w skali 0–256.
Returns
prev przefiltrowana wartość

◆ AccGetData()

void AccGetData ( int16_t out[3])

Funkcja zwracająca dane z akcelerometru.

Pobiera surowe pomiary X/Y/Z z akcelerometru, usuwa offset (kalibrację), a następnie przepuszcza każdą oś przez integer-owy LPF. Wynikiem są trzy wartości całkowite w out[3], gotowe do dalszej obróbki.

Parameters
[in]outtablica danych poddanych filtracji i usunięciu offsetu

◆ accInit()

uint8_t accInit ( void )

Funkcja inicjalizująca działania akcelerometru.

Sprawdza, czy czujnik LSM303C jest podłączony i działa: 1) odczytuje ID akcelerometru i porównuje z oczekiwanym, 2) wywołuje BSP_COMPASS_Init() do inicjalizacji sensora.

Returns
Zwraca 0 przy sukcesie lub kod błędu (1 = niepoprawne ID, 2 = błąd init).
